Test Name User Login and Authentication
Input
  • Login with valid email (testuser@gmail.com) and correct password
  • Login with valid email but incorrect password
  • Login with both email and password fields left empty
Description Testing the login form on Mayurika Jewellery to verify that valid credentials grant access, invalid credentials are rejected with a clear error message, and empty fields trigger client-side validation warnings before any server request is made.
Expected Output
  • Valid credentials: user is redirected to the Home page with their name shown in the navbar
  • Incorrect password: error toast "Invalid email or password" is displayed
  • Empty fields: inline validation messages "Email is required" and "Password is required" appear
Actual Output
  • Valid credentials: user successfully logged in and redirected to Home page.
  • Incorrect password: error toast displayed as expected.
  • Empty fields: validation messages appeared without sending any server request.
Test Case Successful.
